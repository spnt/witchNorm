JS规范 - JS基础规范

概述

本页面为JS基础规范，即只要书写JS代码都应当遵循的规范

命名规则

变量，函数等采用小驼峰式命名法

私有成员或变量在命名前加入下划线_

常量声明须用大写并用下划线分割，如：NAME_LIKE_THIS，并且永远不要使用 const 关键词
枚举型对象：对象名用大写，枚举量用小写

类命名采用大驼峰式命名法

JS文件命名

字母使用小写，若是多个单词用 '-' 或者 英文点号 分割，如：child-process.js，witch.ui.js

变量

声明变量必选添加 var 关键字
定义多变量时，如果某个变量初始化了，就另起一行，重新用一个var来定义余下的变量

  1. var _a, _b, _c = true;
  2. var _i, _b = false;
  3. var _e;

字面量

使用{},[]代替new Object(),new Array()，不要使用string,bool,number对象类型，即不要调用new String,new Boolean,new Number

分号

总是使用分号结尾，分号不能省略

单引号和双引号

请优先使用单引号

空格的使用

数值操作符(如, +/-/*/% 等)两边留空;
赋值操作符/等价判断符两边留一空格;
小括号前后留一个空格, 函数名末尾和左括号之间不要出现空格
for 循环条件中, 分号后留一空格;
变量声明语句, 数组值, 对象值及函数参数值中的逗号后留一空格;
空行不要有空格;
行尾不要有空格;
逗号和冒号后一定要跟空格;
点号前后不要出现空格;
空对象和数组不需要填入空格

缩进

一律使用2空格缩进，如果习惯使用TAB，请将IDE的TAB设置为2个空格
WebStorm设置：File -> Settings -> JavaScript -> Use tab character

去除收尾多余空格

去除收尾多余空格

不要扩展内建类型

不要扩展 javascript 内建对象的方法。
一个错误的写法

  1. Array.prototype.empty = function() {
  2.   return !this.length;
  3. }

使用有意义的判断条件

所有复杂的条件判断都需要赋予一个有意义的名字或者方法

  1. var isValidPassword = password.length >= 4 && /^(?=.*\d).{4,}$/.test(password);
  2. if (isValidPassword) {
  3.   console.log('winning');
  4. }

文件注释

文件注释用来说明当前文件所实现内容，主要包括说明部分和作者部分。
说明部分：描述文件实现的功能、模块或接口等，如有需要增加使用说明或者范例。
作者部分：记录文件的开发者名称和联系方式，用以后续的维护或者交流沟通等。

  1. /**
  2. * ------------------------------------------
  3. * 水平滑动器逻辑封装实现文件
  4. * @version 1.0
  5. * @author 左盐(huabinglan@163.com)
  6. * ------------------------------------------
  7. */

函数注释

函数注释用来描述当前定义的函数的功能及输入输出数据，主要包括说明部分、输入部分、输出部分。
说明部分：说明当前接口的功能，如有需要可增加使用说明或者范例。
输入部分：说明输入数据的类型及含义。
输出部分：说明输出数据的类型及含义。

  1. /**
  2. * 数据是否为整型
  3. * @param d 待校验值
  4. * @param defint 默认值
  5. * @returns {int}
  6. */
  7. tools.check_data.check_int = function(d,defint) {
  8.     if(parseInt(d) != d) return defint;
  9.     return d;
  10. }

块内函数声明

不要在块内声明一个函数，比如：

  1. if(x) {
  2. function foo() {}
  3. }

Array遍历

统一写法

  1. for(var i = 0,l = _array.length; i<l; i++) {
  2.  ......
  3. }

关于==和===

由于==在判断之前会做类型转换，所以尽量使用===和!==进行逻辑判断
